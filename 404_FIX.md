# 404 Not Found Error - Fixed

## Problem
The proxy was returning `404 {"detail":"Not Found"}` when trying to access Terpedia models.

## Root Cause
The `TERPEDIA_BACKEND_URL` environment variable was set to include the full path (`http://104.197.255.123:8010/v1/chat/completions`), but the code was appending `/v1/chat/completions` again, creating an invalid URL:
- Wrong: `http://104.197.255.123:8010/v1/chat/completions/v1/chat/completions` ❌
- Correct: `http://104.197.255.123:8010/v1/chat/completions` ✅

## Solution
1. **Removed** the old environment variable that included the full path
2. **Set** `TERPEDIA_BACKEND_URL` to just the base URL: `http://104.197.255.123:8010`
3. **Updated** the code to properly construct the URL by:
   - Removing any trailing slashes from the base URL
   - Appending `/v1/chat/completions` correctly
4. **Redeployed** the proxy to Vercel

## Code Changes
```typescript
// Before
const terpediaBackendUrl = process.env.TERPEDIA_BACKEND_URL || 'http://104.197.255.123:8010';
const targetUrl = useTerpediaBackend 
  ? `${terpediaBackendUrl}/v1/chat/completions`
  : openaiUrl;

// After
const terpediaBackendBase = process.env.TERPEDIA_BACKEND_URL || 'http://104.197.255.123:8010';
const terpediaBackendUrl = terpediaBackendBase.replace(/\/$/, '');
const targetUrl = useTerpediaBackend 
  ? `${terpediaBackendUrl}/v1/chat/completions`
  : openaiUrl;
```
