# 503 Service Unavailable Error - Fixed

## Problem
LibreChat was returning `503 Service Unavailable` errors for endpoints like `/api/user`, `/api/enable`, `/api/admin`.

## Root Cause
**Meilisearch API Key Mismatch**: 
- The Meilisearch container was using key: `a8786b7e109015396037bd2ef4e770fa3f71100e62dcbf822c8d04f68f387c3d`
- LibreChat's `.env` file had a different key: `DrhYf7zENyR6AlUCKmnz0eYASOQdl6zxH7s7MKFSfFCt`
- LibreChat couldn't authenticate with Meilisearch, causing search/indexing operations to fail
- This caused the API endpoints to return 503 errors

## Solution
1. **Updated** `.env` file to use the same `MEILI_MASTER_KEY` as the Meilisearch container
2. **Restarted** LibreChat service to pick up the new environment variable

## Changes Made
```bash
# Before
MEILI_MASTER_KEY=DrhYf7zENyR6AlUCKmnz0eYASOQdl6zxH7s7MKFSfFCt

# After  
MEILI_MASTER_KEY=a8786b7e109015396037bd2ef4e770fa3f71100e62dcbf822c8d04f68f387c3d
```
